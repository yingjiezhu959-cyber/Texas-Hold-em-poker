# Texas Hold'em Poker｜朋友娱乐局德州扑克

这是一个可以部署到 Render 的网页德州扑克小游戏，支持两人局和三人局。玩家打开同一个网页，通过“房间号”进入同一张牌桌，凑满人数后自动发牌。

> 重要说明：本项目是“娱乐局 / 非公平牌局”，只适合朋友之间使用虚拟筹码整活，不用于真钱赌博、竞赛或任何严肃牌局。

## 功能

- 支持两人对战 / 三人对战
- 房间号创建和加入
- 邀请链接自动带房间号和人数
- 每位玩家只看到自己的底牌
- 本手结束后公开所有玩家底牌
- 支持弃牌、过牌 / 跟注、加注、全下
- 支持边池和摊牌结算
- 支持服务器实时推送状态，减少手动刷新和延迟
- 页面明确标识“娱乐局 / 非公平牌局”
- 内置娱乐码输入区，适合朋友局彩蛋玩法

## 游戏规则说明

### 1. 基础目标

每位玩家拿 2 张底牌，桌面最多发出 5 张公共牌。玩家用自己的 2 张底牌和 5 张公共牌中任选组合，组成最强的 5 张牌。

一手牌可能通过两种方式获胜：

- 其他玩家全部弃牌，你直接赢得底池。
- 进入摊牌，牌型最大者赢得底池；如果牌力相同，则平分底池。

### 2. 本项目牌型大小

从大到小：

1. 同花顺
2. 四条
3. 葫芦
4. 同花
5. 顺子
6. 三条
7. 两对
8. 一对
9. 高牌

### 3. 筹码和盲注

- 初始筹码：1000
- 小盲：5
- 大盲：10
- 如果有玩家筹码归零，下一手开始时会自动恢复到初始筹码，方便朋友娱乐局继续玩。

### 4. 两人局流程

两人局采用 Heads-up 规则：

- 庄家同时是小盲。
- 另一位玩家是大盲。
- 翻牌前，小盲先行动。
- 翻牌后，大盲先行动。
- 每手结束后，庄家位置轮换。

### 5. 三人局流程

三人局按常规按钮位流程：

- 庄家左手边是小盲。
- 小盲左手边是大盲。
- 翻牌前，大盲左手边玩家先行动。
- 翻牌后，庄家左手边第一个仍在牌局中的玩家先行动。
- 每手结束后，庄家位置轮换。

### 6. 每手牌阶段

一手牌按以下阶段进行：

1. 发底牌：每人 2 张。
2. 下盲注：小盲 5，大盲 10。
3. 翻牌前下注。
4. 翻牌：发 3 张公共牌。
5. 转牌：发第 4 张公共牌。
6. 河牌：发第 5 张公共牌。
7. 摊牌结算。

如果中途只剩一名未弃牌玩家，则立即结算，不再继续发牌。

### 7. 玩家操作

- 弃牌：放弃本手牌。
- 过牌：当前无人下注时，不投入筹码，把行动交给下一位。
- 跟注：当前有人下注时，补齐到当前最高下注额。
- 加注：在当前最高下注额基础上继续提高下注。
- 全下：把自己剩余筹码全部投入底池。
- 下一手：本手结束后，入座玩家可以开始下一手。

## 如何在线玩

1. 打开部署后的网页链接。
2. 第一个玩家选择“两人对战”或“三人对战”。
3. 输入房间号，例如 `room1`。
4. 点击“加入 / 创建房间”。
5. 复制邀请链接发给其他玩家。
6. 凑满人数后自动发牌。

同一个房间号就是同一张牌桌。不同朋友局建议使用不同房间号。

## 本地运行

本项目只依赖 Python 标准库，不需要额外安装包。

```bash
python server.py --host 127.0.0.1 --port 8010
```

然后打开：

```text
http://127.0.0.1:8010
```

也可以运行自测：

```bash
python server.py --self-test
```

## 部署到 Render

本仓库已经包含 `render.yaml`，可以直接部署到 Render。

### 方法一：Blueprint 部署

1. 打开 [Render Dashboard](https://dashboard.render.com/)。
2. 选择 New。
3. 选择 Blueprint。
4. 绑定本 GitHub 仓库。
5. Render 会读取 `render.yaml` 并创建 Web Service。
6. 等部署完成后，Render 会生成一个固定访问链接。

### 方法二：手动创建 Web Service

如果不用 Blueprint，也可以手动创建：

- Runtime: Python
- Build Command:

```bash
pip install -r requirements.txt
```

- Start Command:

```bash
python server.py --host 0.0.0.0 --port $PORT
```

Render 会自动提供 `$PORT`，不要写死端口号。

## Render 免费版注意事项

Render 免费版可以跑这个项目，但有几个限制：

- 空闲一段时间后服务会休眠。
- 休眠后第一次打开会比较慢。
- 服务重启后，房间状态会清空。

如果想长期稳定玩，建议升级到付费实例。
